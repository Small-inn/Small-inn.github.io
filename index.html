Angular JS 介绍

1.定义：是一款前端的MVC框架
2.与jQuery的区别：jQuery是一个类库（一系列函数的集合）以DOM为驱动（核心），
  而Angular是一个框架（一系列的类库的集合），以数据和逻辑为驱动

3.AngularJS核心：模块化、双数据绑定、语义化标签、依赖注入

4.与之相似的还有React、vue等框架

MVC

MVC是一种开发模式（Model、View、Controller）
模型：一般用来处理数据（读取/设置）,一般操作数据库
视图：一般用来展示数据,听过HTML展示
控制器：一般是用作链接模型和视图的桥梁

模块化

AngularJS构建应用时是以模块化的方式组织的，
将整个应用化成若干模块，各个模块都有自己的各自职责，然后组成一个整体


定义应用:ng-app属性，表示此标签所包裹的内容都属于App的一部分

定义模块：AngularJS提供一个全局对象angular，此全局对象存在若干方法， 、
其中，angular.module()方法定义一个模块（其实App本质是一个模块大的模块）

定义控制器：作为链接模型和视图的桥梁存在，所以当定义好了控制器以后也就定义好了模型和视图

模型数据是要展示导视图上的，所以需要将控制器关联到视图上，
通过HTML标签加ng-controller属性并且赋值响应的控制器的名称，确定关联关系

具体操作：
第一步：引入angular包
第二步：创建模板
第三步：创建自己的代码

var App = angular.module('App',[])

App.controller('Demo',['$scope',function($scope){
    //自己的代码
    $scope.name = '客店小二LX'

}])

body ng-app = 'App'

div ng-constroller='Demo'

h1 {{name}}

指令

1.ng-app：指定一个应用
可以应用任意标签上，并将此标签区视为一个应用的范围，
一个页面可以有多个但是不能嵌套，一般制定一个

2.ng-controller：指定某视图隶属某一个控制器
var App = angular.mudule(‘App’,[]);

App.controller(‘DemoController’,[$scope,function($scope){

}])

3.ng-show:当属性值为true时显示，false时不显示
4.ng-hide:当属性值为true时隐藏，false时显示
5.ng-if:当属性值为true时显示，false时不显示
5.ng-src:增强地址
6.ng-href增强地址
7.ng-class控制类名
8.ng-include引入模板
9.ng-disabled表单禁用
10.ng-readonly表单只读
11.ng-checked单/复选框表单选中
12.ng-selected下拉框表单选中
13.自定义指令
通过angular全局对象下的directive方法实现

// 自定义控制器
App.controller('ItcastController', ['$scope', function ($scope) {

    $scope.list = ['html', 'css', 'js'];

// 自定义指令

// 第二个参数，回调函数
App.directive('itcast', function () {
    // 此回调函数指定具体指令应该做哪些事情
    // 通过一个对象来定义

    var obj = {
        // 自定义指令可以是属性可以是标签
        // E element
        // A attribute
        // C class
        // M mark
        restrict: 'ECMA',
        // 所有内容必须包含在一个根元素下
        template: '<div><h1>你好, AngularJS!</h1><p>哈哈</p></div>',
        // templateUrl: './header.html',
        replace: true,
        // scope: {},
        // link: function () {},
        // compile: function () {}
    }

    return obj;
});

typeof关键字

  • 返回值：字符串类型
  • typeof获取不到详细的类型，只能获取到简单的数据类型。
  • 使用typeof获取复杂类型只能取到 object
    //如何获取详细类型
    Object.prototype.toString 方法可以将对象转化成字符串
    格式：[object 详细的字符串类型]
    注意：一个对象的toString方法 返回的是[typeof获取到的类型 构造函数的名称]

对象的动态特性

第四天复习（作用域链与变量提升）

递归：

定义：在函数内部直接或者间接的调用自己，如果没有结束条件，就是死递归，所以递归要有两个条件（1.自己调用自己 2.要有结束条件）

作用域

  • 定义：指的是变量起作用的范围，在JS中有且只有函数可以创建作用域

  • 词法作用域：上一旦代码写好，就可以从代码的结构中分析出，变量的作用域，这种作用域就是词法作用域；

变量的提升

  • js执行分两个阶段：
    1.预解析：就是将函数，以及变量的声明提升到其所在的作用域的最顶上
    2.按从上到下执行代码

  • 注意： 1.函数声明会被提升
    2.函数表达式不会被提升
    3.new Function 这种形式创建的函数，不能被提升

  • 同名注意事项：
    当函数同名的时候，汇都提升，但是后面的会把前面的给覆盖掉
    当函数名与变量名同名的时候，会把函数提升，忽略变量

  • js中的变量和函数提升是分段的；

  • 绘制作用域链步棸；
    1.先绘制一条直线，表示0级作用域
    2.在全局作用域中，查找变量以及函数的声明，一小方块形式表示，画在0级作用域链上
    3.从0级作用域链中函数中引出新的直线，表示1级作用域链
    4.再去对应的函数中查找变量以及函数的声明，一小方块形式，画在1级作用域链上
    5.再沿着当前作用域往里找，找到函数画一条直线，一直到找不到为止

  • 变量搜索原则
    1.当遇到一个变量的使用的时候，先在当前的作用域中去找这个变量，如果找到了，就直接使用
    2.如果没有找到，就去外层的作用域中进行查找，如果找到了，就直接使用
    3.如果没有找到，就继续往上查找，直到查找到0级作用域链
  • 1.in关键字 用来判断是否可以通过对象来访问每个属性
    2.声明全局变量，就相当于给window对象新增一个属性
